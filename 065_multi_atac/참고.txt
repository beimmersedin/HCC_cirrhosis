  ---
  A. 구조적 문제 (심각도 순)

  1. Sample-level confounding — 가장 치명적

  Cirrhosis = C1_TC (1 patient)
  Hepatitis = NC1_TC + NC2_TC (2 patients)
  Hepatocytes 175 vs 2,559 세포라고 했지만, Cirrhosis가 사실상 1명의 환자입니다. 현재 파이프라인의 689 DARs는 "Cirrhosis effect"가
  아니라 "C1 환자 vs 나머지 환자" 차이일 가능성이 큽니다. 이건 HNF4A 결론 자체를 뒤흔드는 문제입니다.

  논문에서의 표준 대응:
  - Pseudobulk DESeq2/edgeR로 sample-level 통계를 적용 (n=1 vs n=2라 사실상 불가)
  - 최소한 이 limitation을 명시적으로 논의
  - GeoMx spatial data로 독립 검증 (이미 데이터가 있으므로)

  2. Motif enrichment 필터가 실제로 적용되지 않음

  TF_Hepatocyte_Up.R:139-142에서 291개 enriched motif 필터링이 주석 처리되어 있습니다:
  # sig_motifs <- enriched_motifs$motif[enriched_motifs$p.adjust < 0.05]  # 291
  # peak_tf_mapping <- peak_tf_mapping[peak_tf_mapping$motif_id %in% sig_motifs, ]
  결과적으로 Peak_TF_Match_Map에는 692개 전체 motif의 51,579 pairs가 들어갑니다. Summary에는 "291 enriched motifs"라고 쓰지만, 실제
  downstream에는 enrichment 통과하지 못한 motif도 포함됩니다. Step 1의 enrichment test가 사실상 장식입니다.

  3. 5개 cell type 분석이 copy-paste로 분리됨

  TF_Hepatocyte_Up.R, TF_Macrophages_Up.R, TF_T_Cells_Up.R 등이 cell type 이름만 바꾼 동일 코드입니다. LinkPeaks_*.R과 Linkage_*.R,
  GRN_*.R도 마찬가지. 이건 구조적 문제를 넘어 유지보수와 재현성 문제입니다. 하나의 parameterized script로 통합해야 합니다.

  4. Tier1 integration이 Hepatocytes에만 존재

  5개 cell type 모두 DA→TFmotif→Linkage→GRN까지 돌렸는데, Tier1 causal integration은 Hepatocytes만 했습니다. 나머지 4개의 GRN 결과가
  공중에 떠 있습니다.

  ---
  B. 불필요하거나 중복된 단계

  1. Linkage_*.R과 LinkPeaks_*.R의 이중 구조가 불필요

  Linkage_Hepatocyte.R은 단순히 두 CSV를 merge하는 50줄짜리 스크립트입니다. LinkPeaks 스크립트 끝에 merge 코드를 붙이면 됩니다.

  2. chromVAR를 Up/Down 별도로 실행할 필요 없음

  chromVAR는 전체 peak matrix 기반이므로 Up/Down에서 결과가 동일합니다. Down-DAR은 2개뿐이라 enrichment도 의미 없습니다.

  3. Final_Weighted_GRN.csv는 Tier1에서 사용되지 않는 중간 산출물

  4. pipeline_overview_figure.py와 RE_pipeline_overview_figure.py가 중복

  ---
  C. 방법론적 문제

  1. DA에서 Logistic Regression 대신 pseudobulk이 필요

  Cell-level LR은 같은 sample 내 세포들의 비독립성을 무시합니다. 2022년 이후 논문에서는 pseudobulk (DESeq2/edgeR)가 표준입니다.

  2. GRN의 Spearman ρ > 0.1 기준이 너무 낮음

  0.1은 거의 노이즈 수준입니다. 논문에서는 최소 ρ > 0.2~0.3, 또는 permutation-based null distribution 사용.

  3. "Causal" 용어가 과도함

  파이프라인 전체가 correlative evidence입니다. "Causal Evidence", "Causal GRN" 네이밍은 reviewer에게 즉시 공격 대상이 됩니다.

  권장 용어:
  - "Causal" → "Multi-evidence regulatory" 또는 "Integrative evidence"
  - "Causal Evidence Score" → "Regulatory confidence score"

  4. Evidence score normalization이 arbitrary

  ev_dar_sig = pmin(-log10(dar_padj + 1e-300) / 30, 1)  # 왜 30?
  ev_dar_fc  = pmin(abs(dar_log2FC) / 5, 1)              # 왜 5?
  ev_pwm     = pmin(pwm_score / 20, 1)                    # 왜 20?
  cap 값들의 근거가 없습니다. Quantile 기반이나 rank-based normalization이 더 견고합니다.

  5. Geometric mean의 zero-inflation 문제

  하나라도 0이면 전체가 0. PWM score가 0인 진짜 regulatory edge가 탈락될 수 있습니다.
